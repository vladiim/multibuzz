mbuzz · onboarding cohesion — every screen in one place
Every onboarding screen wrapped in the same chrome, the same body archetypes, the same typography, the same button placement. Same primary action position (bottom right). Hero screens centered, task screens left-aligned. Cards share one border/padding pattern. Pricing pitch lives in the sanctioned indigo promo card (used on install_service only).

Body archetypes in use:
  • Chooser — setup-choice. Centered hero heading + horizontal row of N tappable cards.
  • Explainer — install_service. Centered title + numbered card grid + sanctioned promo + risk-reversal strip + bottom-right Continue.
  • Form — discovery, guided_setup, invite_teammate, payment_setup. Left-aligned heading + cards containing labeled fields/options + bottom-right primary CTA. Optional left-aligned Back.
  • Waiting — verify, payment_complete (processing). Centered spinner + heading + helper copy. Auto-advances.
  • Success — attribution, payment_complete (in_progress). Centered hero emoji + heading + body + single primary CTA.
  • Setup grid — self-serve setup (API key + SDK). Same form archetype but with mixed content blocks: code panel + card grid.
